# User Stories

## Pendahuluan

User stories digunakan untuk menjelaskan kebutuhan pengguna berdasarkan sudut pandang masing-masing role dalam aplikasi FoodCycle App. Role utama dalam aplikasi ini adalah pembeli, penjual, dan admin.

## User Stories Pembeli

| Fitur | User Story |
|---|---|
| Registrasi dan Login | Sebagai pembeli, saya ingin membuat akun dan login agar dapat mengakses aplikasi. |
| Melihat Daftar Produk | Sebagai pembeli, saya ingin melihat daftar makanan surplus yang tersedia agar dapat memilih makanan yang ingin dibeli. |
| Melihat Detail Produk | Sebagai pembeli, saya ingin melihat detail produk agar mengetahui informasi seperti nama produk, harga, jumlah, lokasi, dan waktu kedaluwarsa. |
| Memesan Produk | Sebagai pembeli, saya ingin memesan produk agar dapat membeli makanan surplus sebelum produk tersebut tidak tersedia. |
| Checkout Sederhana | Sebagai pembeli, saya ingin melihat ringkasan pesanan sebelum melakukan konfirmasi agar dapat memastikan pesanan sudah sesuai. |
| Konfirmasi Pesanan | Sebagai pembeli, saya ingin mengonfirmasi pesanan agar transaksi dapat diproses oleh penjual. |

## User Stories Penjual

| Fitur | User Story |
|---|---|
| Registrasi dan Login | Sebagai penjual, saya ingin membuat akun dan login agar dapat mengelola produk makanan surplus. |
| Unggah Produk | Sebagai penjual, saya ingin mengunggah produk makanan surplus agar makanan yang masih layak konsumsi dapat dijual sebelum terbuang. |
| Edit Produk | Sebagai penjual, saya ingin mengedit informasi produk agar data produk tetap akurat. |
| Hapus Produk | Sebagai penjual, saya ingin menghapus produk yang sudah tidak tersedia agar pembeli tidak memesan produk yang habis. |
| Dashboard Penjual | Sebagai penjual, saya ingin melihat daftar produk yang telah saya unggah agar dapat memantau produk yang tersedia. |
| Melihat Pesanan | Sebagai penjual, saya ingin melihat pesanan yang masuk agar dapat memproses pesanan pembeli. |

## User Stories Admin

| Fitur | User Story |
|---|---|
| Verifikasi Penjual | Sebagai admin, saya ingin memverifikasi akun penjual agar penjual yang menggunakan aplikasi dapat lebih terpercaya. |
| Melihat Data Produk | Sebagai admin, saya ingin melihat data produk yang tersedia agar dapat memantau aktivitas pada aplikasi. |
| Melihat Laporan | Sebagai admin, saya ingin melihat laporan jumlah produk dan pesanan agar dapat mengetahui perkembangan aplikasi. |

## Prioritas Fitur

## Fitur MVP

Fitur yang menjadi prioritas utama dalam pengembangan awal adalah:

- Registrasi dan login
- Unggah produk makanan surplus
- Melihat daftar produk
- Melihat detail produk
- Pemesanan produk
- Checkout sederhana
- Dashboard penjual

## Fitur Pengembangan Selanjutnya

Fitur yang akan dikembangkan pada tahap selanjutnya adalah:

- Verifikasi akun penjual
- Tracking pesanan secara detail
- Integrasi pembayaran online
- Dashboard laporan admin
