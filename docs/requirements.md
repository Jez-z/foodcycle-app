# System Requirements

## Functional Requirements

| Code | Requirement |
|---|---|
| FR-01 | The system allows users to register and login |
| FR-02 | The system allows sellers to upload surplus food products |
| FR-03 | The system allows buyers to view the list of available products |
| FR-04 | The system allows buyers to view product details |
| FR-05 | The system allows buyers to order products |
| FR-06 | The system allows buyers to complete a simple checkout process |
| FR-07 | The system allows sellers to view their uploaded products through a dashboard |

## Non-Functional Requirements

| Code | Requirement |
|---|---|
| NFR-01 | The application should be easy to use |
| NFR-02 | The application should have a simple and responsive interface |
| NFR-03 | The application should be accessible through a web browser |
| NFR-04 | The application should display product data clearly |
| NFR-05 | The application should support basic user interaction smoothly |

## MVP Scope

The MVP focuses on the main marketplace flow:

1. Seller uploads surplus food product
2. Buyer views available products
3. Buyer opens product detail
4. Buyer orders the product
5. Buyer completes simple checkout
6. Seller monitors products through dashboard

## Future Development Scope

The following features are planned for future development:

- Online payment integration
- Detailed order tracking
- Seller account verification
- Admin report dashboard
