# Kebutuhan Sistem

## Kebutuhan Fungsional

| Kode | Kebutuhan |
|---|---|
| FR-01 | Sistem memungkinkan pengguna untuk melakukan registrasi dan login |
| FR-02 | Sistem memungkinkan penjual untuk mengunggah produk makanan surplus |
| FR-03 | Sistem memungkinkan pembeli untuk melihat daftar produk yang tersedia |
| FR-04 | Sistem memungkinkan pembeli untuk melihat detail produk |
| FR-05 | Sistem memungkinkan pembeli untuk melakukan pemesanan produk |
| FR-06 | Sistem memungkinkan pembeli untuk melakukan checkout sederhana |
| FR-07 | Sistem memungkinkan penjual untuk melihat produk yang telah diunggah melalui dashboard |

## Kebutuhan Non-Fungsional

| Kode | Kebutuhan |
|---|---|
| NFR-01 | Aplikasi mudah digunakan oleh pengguna |
| NFR-02 | Aplikasi memiliki tampilan yang sederhana dan responsif |
| NFR-03 | Aplikasi dapat diakses melalui web browser |
| NFR-04 | Aplikasi dapat menampilkan data produk dengan jelas |
| NFR-05 | Aplikasi mendukung interaksi dasar pengguna dengan lancar |

## Ruang Lingkup MVP

MVP berfokus pada alur utama marketplace makanan surplus:

1. Penjual mengunggah produk makanan surplus
2. Pembeli melihat produk yang tersedia
3. Pembeli membuka detail produk
4. Pembeli melakukan pemesanan
5. Pembeli melakukan checkout sederhana
6. Penjual memantau produk melalui dashboard

## Ruang Lingkup Pengembangan Selanjutnya

Fitur berikut direncanakan untuk pengembangan selanjutnya:

- Integrasi pembayaran online
- Tracking pesanan secara detail
- Verifikasi akun penjual
- Dashboard laporan admin
